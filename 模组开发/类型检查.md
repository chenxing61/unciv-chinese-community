# 类型检查

错误是难免的。命名字段、忘记添加东西，甚至不知道存在的东西。计算机可以自己处理很多这些问题，所以我们可以让它们做这项工作，以确保我们的 JSON 文件是正确的，方法是使用*JSON Schemas*。

JSON Schema 文件还启用自动完成、警告和一些智能感知，直接在你的编辑器中。

## Android Studio

1. 双击空格，搜索"json schema mappings"，进入
2. 点击小'+'（顶部，'language & frameworks'文本下）
3. 将 URL 放为 `https://raw.githubusercontent.com/yairm210/Unciv/master/docs/Modders/schemas/Buildings.schema.json`
4. 点击'Schema version'文本下的'+'，添加'File pattern'，将模式设为 `*/Buildings.json`

Android Studio 现在将识别所有 Buildings.json 文件属于该模式，并警告不一致。

## VSCode 或类似

有两种方法可以在 VSCode 中启用 JSON Schemas：通过安装 [VSCode Unciv Extension](https://github.com/robloach/vscode-unciv)，或手动添加它们。使用扩展更容易。

### 使用扩展

[VSCode Unciv Extension](https://github.com/robloach/vscode-unciv) 启用 JSON Schemas，设置正确的文件关联，以及其他一些功能。

1. 打开 VSCode
2. 按 *CTRL+P* 打开命令面板
3. 编写以下内容，然后按 Enter：
   ```
   ext install robloach.unciv
   ```
4. 恭喜，你现在安装了 Unciv JSON Schema 文件，具有自动完成、智能感知和文件检查。

### 手动配置

如果你不想使用 VSCode 扩展，也可以手动启用 Schemas...

## 在线工具

如果你不使用任何这些编辑器，你可以使用[此工具](https://www.jsonschemavalidator.net/)在线检查你的文件。但是，它无法处理 VSCode 和 Android Studio 处理的缺失逗号，所以你可能需要让你的 json 符合规范才能使用它。

你要验证的模式是：

```json
{
    "$ref": "https://raw.githubusercontent.com/yairm210/Unciv/master/docs/Modders/schemas/Buildings.schema.json"
}
```

## 支持的 JSON 文件

以下是 Unciv 模组中可以使用的 JSON 文件及其对应的 Schema：

### 文明相关

- Beliefs.json - 信仰
- Buildings.json - 建筑
- CityStateTypes.json - 城邦类型
- Nations.json - 文明
- Policies.json - 政策
- Quests.json - 任务
- Religions.json - 宗教
- Specialists.json - 专家

### 地图相关

- Terrains.json - 地形
- TileResources.json - 地块资源
- TileImprovements.json - 地块改良
- Ruins.json - 遗迹

### 单位相关

- Units.json - 单位
- UnitPromotions.json - 单位晋升
- UnitTypes.json - 单位类型
- UnitNameGroups.json - 单位名称组

### 杂项

- Difficulties.json - 难度
- Eras.json - 时代
- ModOptions.json - 模组选项
- GlobalUniques.json - 全局 Uniques
- Speeds.json - 游戏速度
- Tutorials.json - 教程
- VictoryTypes.json - 胜利类型

## 常见错误

### 缺少字段

如果你看到"Missing required property"错误，这意味着你缺少某个必需的字段。检查 Schema 以查看哪些字段是必需的。

### 类型错误

如果你看到"Type mismatch"错误，这意味着你使用了错误的数据类型。例如，在应该是数字的地方使用了字符串。

### 无效值

如果你看到"Value is not one of the allowed values"错误，这意味着你使用的值不在允许的值列表中。

### 额外字段

如果你看到"Additional properties not allowed"错误，这意味着你添加了一个不被允许的字段。

## 最佳实践

1. **使用编辑器支持**：使用支持 JSON Schema 的编辑器（如 VSCode 或 Android Studio）可以获得自动完成和实时错误检查。

2. **定期检查**：定期使用在线工具或游戏内的验证器检查你的模组。

3. **测试你的模组**：即使没有 Schema 错误，也要在游戏中测试你的模组，以确保它按预期工作。

4. **查看错误消息**：如果遇到错误，仔细阅读错误消息，它们通常会告诉你问题所在。

5. **使用示例**：参考基础游戏或其他模组中的示例文件，以了解正确的格式。
