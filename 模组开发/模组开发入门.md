# 模组开发入门

## 什么是模组？

每个人都有自己想要在游戏中实现的想法。遗憾的是，游戏只能理解代码，所以模组是我们为那些不会编码的人提供一定自由度的方式。

模组可以**添加、替换和删除**基本的游戏定义，例如单位、文明、建筑、改良、资源和地形。

加载了这些模组的游戏将按照模组的定义运行。

游戏只能识别现有的定义，所以你不能为文明/单位/建筑等**添加新的独特能力**，只能使用现有的能力进行组合。

## 模组的主要类型

### 扩展模组

这些模组向基础规则集添加新的文明/单位/建筑/资源——可以是默认规则集，也可以是基础规则集模组。这很容易实现，也是开始的好地方。

**示例**：创建一个新文明

### 基础规则集模组

这些模组替换整个现有的规则集——科技树、单位、政策、文明等——提供与基础游戏完全不同的体验。这些通常需要相当多的工作，但能提供全新的体验，因此也是最受欢迎的。

**要求**：参见[基础规则集要求](mod-file-structure/overview.md#基础规则集要求)

### 规则集无关模组

这些模组不包含任何规则集相关的 JSON 文件，而是包含其他效果。视听模组（包括地形集、单位集和 UI 皮肤）和地图模组属于此类。

## 模组命名

模组需要符合 GitHub 仓库命名规则，但最好保持简单，只使用字母、数字和连字符 `-`。

连字符会**自动**转换为空格，用于 Unciv 中的显示和使用。

许多标点符号或扩展 Unicode 字符**可能**可以使用，但充其量会让潜在用户觉得不吸引人，最坏的情况下，我们可能会拒绝支持遇到问题的你。

## 模组组件

模组位于 `/mods` 目录中。在桌面端，该目录应该与 `.jar` 文件相邻。

模组通常有 2 个子文件夹：

- `jsons` - 在这里放置改变游戏对象数据的文件，文件顺序与基础游戏中的顺序相同。更多信息参见[Mod 文件结构详解](Mod 文件结构详解.md)
- `Images` - 在这里放置游戏图像，如基础图像文件所示

要删除游戏中的对象，你需要在 `/jsons` 子文件夹中创建一个 ModOptions 文件。

**基础规则集模组**是"从头开始"的模组——所有原始对象都被删除，只使用该特定模组的对象。

这是通过在 ModOptions 文件中添加 `"isBaseRuleset":true` 配置来实现的。详见[杂项 JSON 文件说明](杂项 JSON 文件说明.md)。

## 视听组件

除了更改规则——甚至不更改规则——模组可以覆盖现有的图形或声音，或添加音乐曲目。详情参见[视听模组](图像和音频.md)。

自定义地形集和单位集是这些模组的子组——参见[创建自定义地形集](自定义地形集.md)——UI 皮肤模组也是如此，参见[创建 UI 皮肤](创建 UI 皮肤.md)。

这些模组是模组管理屏幕上可用的"永久视听模组"开关的候选者，参见[永久视听模组](images-and-audio.md#永久视听模组)。

图像需要在游戏使用之前"打包"，桌面版本可以为你完成此操作。请务必阅读[纹理集](images-and-audio.md#图像和纹理集)章节！

## 向模组添加地图

你也可以向模组添加地图，这样下载你的模组的玩家就可以使用这些地图。

如果只想分享地图，模组也可以仅包含地图。

在地图编辑器中完成地图制作后，保存它，它会出现在游戏的 `/maps` 文件夹中。

将其复制到模组中的 `/maps` 文件夹，就完成了！

## ## 翻译

就像 Unciv 本身一样，模组可以翻译成 Unciv 核心支持的任何或所有语言。

翻译通过游戏内置的翻译系统实现，使用 `.properties` 文件。

## 发布你的模组

为了让任何人都能下载你的模组，你需要创建一个 GitHub 仓库。

Images 和 jsons 文件夹需要位于仓库的根目录。

然后你可以从 Unciv 的模组管理器中手动下载模组：

- 从 Unciv 主屏幕，点击"Mods"
- 点击"从 URL 下载模组"，输入你的 GitHub 页面位置
- 游戏会自动下载并解压你的模组，准备就绪！

测试你的模组**可以**被下载，并且下载后工作良好后，你就准备好进入最后阶段了——**让用户自动获取模组**。

为此，你需要：

- 进入你的 GitHub 页面
- 点击 About 右上角旁边的齿轮图标（页面右上部分）
- 在"Topics"中，添加"unciv-mod"

可选地添加以下一个或多个主题，将你的模组标记为属于特定类别：

- `unciv-mod-rulesets`（用于基础规则集模组）
- `unciv-mod-expansions`（用于扩展原版规则集的模组——请使用这个，**不是** unciv-mod-expansion）
- `unciv-mod-graphics`（用于更改图形的模组——图标、肖像、地形集）
- `unciv-mod-audio`（用于提供音乐或修改声音的模组）
- `unciv-mod-maps`（用于包含地图的模组）
- `unciv-mod-fun`（主要用于调整机制或其他游戏玩法的模组）
- `unciv-mod-modsofmods`（用于扩展另一个模组规则集的模组）

当你打开 Unciv 的模组管理器时，它会查询 GitHub 的带有该主题的仓库列表，现在**你的**仓库也会出现在那里！

类别将作为模组按钮上的注释显示，用户可以过滤它们。游戏使用内容不需要这些类别——例如，你仍然可以从缺少 unciv-mod-maps 主题的模组加载地图。

如果你希望游戏支持新的类别，GitHub 会接受任何主题，但你需要要求 Unciv 团队在游戏中启用它们。

如果你认为应该支持游戏中额外的主题，那么行动过程如下：

- 你可以按照自己的意愿在仓库中添加主题，但希望它们成为 Unciv 支持的主题，必须以"unciv-mod-"开头。
- 完成后，你可以：
  - 等待至少一个版本，检查你的主题是否出现在游戏中
  - 或者，向 Unciv 团队提交请求，说明该主题为什么有利于整个社区。

## 从其他来源加载模组

模组管理器有一个"从 URL 下载模组"按钮。作为模组消费者，你可以使用它从你信任的源加载 zip 文件模组：用例包括开发中的模组、出于各种原因无法连接到 GitHub 的客户端（例如防火墙、不支持 IPv4），或者作为在你自己的移动设备上加载自己模组的替代传输方式。

此下载器支持简单的重定向，但不支持需要身份验证或预设 cookie 的托管商，或使用 JavaScript 在浏览器中重新组装文件或其他复杂问题的托管商——简单的 `http get` 就足够了。

作为模组作者，你可能需要了解该按钮如何工作的一些细节。毕竟，它将缺乏通常可以从 GitHub 获取的一些元数据——例如仓库名称、分支名称、发布标签。你的存储可能是替代的 Git 平台，但除非我们编写显式支持，否则 Unciv 无法可靠地确定这些。因此请注意：

- 解压后的 zip 内容应该只包含一个子文件夹，以及该文件夹下的所有模组文件（以及模组文件夹如 `jsons` 或 `Images`），或者在 zip 内容的顶层直接包含所有模组数据。
- 内容应包含 Unciv 知道是模组一部分的大多数文件或文件夹。添加比实际负载文件更多的测试、评论、资产源或类似文件/文件夹将导致"无效的模组存档结构"消息，你的模组将不被接受。这是在顶层模组上计算的，所以当你接近该限制时，将所有额外文件移动到一个子文件夹应该可以工作。请注意，名为"license"、"contribute.md"、"readme.md"或"credits.md"的文件**确实**算作"良好"内容。
- 最终的模组名称将从以下三个来源之一获取：下载链接路径的最后一部分、服务器发送的 content-disposition 标头（你可以直接在浏览器中下载链接时看到它将被保存的实际文件名），或上述第一个子文件夹。优先级是子文件夹 - 标头 - 路径，有一些罕见的例外（例如，当子文件夹名称包含在标头/路径名称中但更无聊时，或者在某些情况下，典型的分支名称后缀会自动删除）。
- 因此，你能做的最好的事情是在单个子文件夹下包含你想要的模组名称（包括正确的大小写）的模组内容，或者如果你的平台使这变得困难，请确保下载的名称与你想在模组列表中显示的匹配（在所有情况下用破折号替换空格是允许的，并且是推荐的）。

## 我有了模组，现在怎么办？

模组的主要用途是在开始新游戏或配置地图时添加它们。这意味着该特定游戏/地图将使用模组的规则集和图像。

对于主要是视觉或音频的模组，还有第二种用途——通过模组管理器，你可以将它们启用为**永久视听模组**。这意味着模组的图像和/或声音将在游戏中替换原始媒体，包含的音乐将可用——参见[这里](images-and-audio.md#提供额外音乐)。

## 手动加载模组的模组位置

一般来说，你永远不应该手动加载你的模组——这不仅笨拙，而且更容易出错。除非你有非常特殊的用例，否则你**不应该这样做**。

加载模组时，它需要位于 `/mods` 中自己的文件夹中——这就是你在编辑模组时的工作方式。

在 Android 上，你可以将它们复制到 `Android/data/com.unciv.app/files/mods` 目录中。

应用启动时，它们会自动复制到 `/data/data/com.unciv.app/files/mods` 目录中，该目录对用户不可访问。

在 Chromebook 上，进入"Play files"，应该在窗口左侧边栏"My files"下。点击窗口右上角"X"下方的 3 个垂直点。
如果选项"显示所有 Play 文件夹"旁边没有勾选，请点击它。你应该会看到屏幕上出现一些新文件。**现在**导航到 `Android/data/com.unciv.app/files/mods`
