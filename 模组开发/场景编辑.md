# 场景编辑

场景是特定的游戏状态，设置为让玩家有特定的体验。

这些范围可以只是在特定位置有城市和单位，到拥有完整的自定义规则集来支持它们。

创建模组时，我们区分**规则集**和**场景**——场景只是特定的游戏状态，换句话说——保存的游戏。

## 创建场景

要创建场景：

- 创建一个你想要的游戏，**以及一个旁观者**
- 作为旁观者进入游戏，使用控制台编辑保存
- 保存游戏，将游戏保存文件复制到模组中的"scenarios"文件夹

## 控制台

### 打开

要从世界屏幕打开控制台，点击键盘上的 `~` 按钮。

在移动设备上：

- 长按菜单汉堡菜单（3 行）
- 点击"开发者控制台"按钮

要查看可用命令，按 Enter。这也适用于子命令（例如，当你输入 `tile` 时）。

### 命令格式

对象名称（单位、建筑、文明等）不区分大小写。

名称中有空格的单位，如"Great General"，可以以 2 种方式输入：

- **"great general"** - 用引号括起来
- **great-general** - 用破折号代替空格

### 自动完成

控制台具有自动完成功能——对于命令和参数使用 Tab。

- 当两个或多个选项匹配时，显示可能性，并为你输入最长的公共子字符串（例如，你输入'c'，Tab：命令'city'或'civ'匹配，控制台部分完成'ci'给你）
- 当你尚未输入部分文本（控制台输入为空或以空格结尾）时，自动完成将显示所有选项

### 其他

一些命令在地图上你选择的单位或地块上操作。你也可以在控制台打开时这样做。

控制台故意不遵循规则集定义的所有规则——例如，它允许在没有新鲜水的山丘上建造农场，或者在工人上获得机动性晋升。任何意外后果都是你的责任。

你可以使用上/下按钮访问命令历史

## 常用命令

### 地块命令

- `tile info` - 显示地块信息
- `tile set terrain <terrain>` - 设置地块地形
- `tile set feature <feature>` - 设置地块特征
- `tile set resource <resource>` - 设置地块资源
- `tile set improvement <improvement>` - 设置地块改良
- `tile clear improvement` - 清除地块改良
- `tile set road` - 设置道路
- `tile clear road` - 清除道路

### 城市命令

- `city create <name>` - 创建城市
- `city remove` - 移除城市
- `city add population <amount>` - 增加人口
- `city set population <amount>` - 设置人口
- `city add building <building>` - 添加建筑
- `city remove building <building>` - 移除建筑
- `city add production <building/unit>` - 添加生产
- `city set production <building/unit>` - 设置生产

### 单位命令

- `unit create <unit>` - 创建单位
- `unit remove` - 移除单位
- `unit add promotion <promotion>` - 添加晋升
- `unit set promotion <promotion>` - 设置晋升
- `unit set health <amount>` - 设置生命值
- `unit set movement <amount>` - 设置移动力
- `unit set owner <civ>` - 设置所有者

### 文明命令

- `civ add tech <tech>` - 添加科技
- `civ remove tech <tech>` - 移除科技
- `civ add gold <amount>` - 添加金币
- `civ set gold <amount>` - 设置金币
- `civ add culture <amount>` - 添加文化
- `civ set culture <amount>` - 设置文化
- `civ add faith <amount>` - 添加信仰
- `civ set faith <amount>` - 设置信仰

### 游戏命令

- `game set turn <turn>` - 设置回合
- `game set era <era>` - 设置时代
- `game reveal all` - 揭示所有地图
- `game fog all` - 所有地图迷雾

## 场景示例

### 创建历史场景

1. 创建一个有历史文明的游戏（例如罗马、迦太基、埃及）
2. 添加一个旁观者
3. 作为旁观者进入游戏
4. 使用控制台放置历史城市和单位
5. 设置正确的科技、时代和资源
6. 保存游戏并复制到模组的"scenarios"文件夹

### 创建教学场景

1. 创建一个简化游戏
2. 添加一个旁观者
3. 作为旁观者进入游戏
4. 设置特定的教学情况
5. 添加提示或说明（通过控制台或编辑保存文件）
6. 保存游戏并复制到模组的"scenarios"文件夹

## 注意事项

- 控制台命令可以绕过正常的游戏规则
- 确保在发布前测试你的场景
- 场景可能需要特定的模组或规则集才能正常工作
- 复杂的场景可能需要大量的编辑和测试

## 进阶技巧

### 编辑保存文件

除了使用控制台，你也可以直接编辑保存文件（JSON 格式）来创建更复杂的场景。这允许你：

- 设置精确的单位属性
- 自定义外交关系
- 添加特定的游戏状态
- 创建复杂的起始条件

### 使用脚本

对于复杂的场景，你可以考虑创建脚本来自动化重复的任务。虽然 Unciv 目前不支持内置脚本，但你可以：

- 使用外部工具生成场景
- 创建一系列控制台命令
- 使用其他模组作为基础
