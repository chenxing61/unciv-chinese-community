# 图像和音频

## 图像和纹理集

图像需要在游戏使用之前"打包"。这个准备步骤只需要一次（只要原始图形不被更改）。

结果是一个或多个文件对——一个 [png 格式](https://zh.wikipedia.org/wiki/PNG)的纹理和一个对应的[图集](https://zh.wikipedia.org/wiki/Texture_atlas)文件。

如果你有一个单独的 `Images` 文件夹，默认的文件对是 `game.png`/`game.atlas`。

对于你的玩家，单独的图像并不重要——只有合并的图像实际上注册到游戏中，所以你需要将它们包含在你的仓库中并保持最新。

我们仍然建议在模组中包含原始图像，以便其他从源代码运行的开发人员可以访问它们。

包含原始图像后，你可以使用开发环境使用 git，将其链接到你的仓库，同时使用符号链接允许 Unciv 看到模组——并在每次启动时为你打包图集。

如果你在 Android 版本的 Unciv 上开发模组（不推荐！），你将无法直接生成这些打包文件。

### 打包纹理集的方法

- 纹理集**不能在 Android 上打包**（技术原因：TexturePacker 使用 `java.awt` 进行繁重的工作，这在 Android 上不可用）
- 使用你的模组启动桌面版本（你的模组主文件夹是游戏"mods"文件夹的子文件夹，或符号链接到那里）。这使用了[此处记录](https://libgdx.com/wiki/tools/texture-packer)的打包方法
- 你可以请求 Discord 服务器中的人帮助你
- 你可以使用外部工具，[例如 gdx-texture-packer-gui](https://github.com/crashinvaders/gdx-texture-packer-gui)。需要极其小心，确保文件可以被 Unciv 发现，内部相对路径是正确的
- 你可以将[此工作流的副本](https://github.com/ravignir/RekMOD/blob/master/.github/workflows/autoupdate.yml)添加到你的模组中，以便在每次提交到仓库时自动打包图集。它应该放在与示例相同的位置（`.github/workflows/autoupdate.yml`）

### 渲染性能

#### 多个纹理集

如果你的模组有很多图像（或很大的图像），纹理可能会"溢出"到额外的纹理文件中——2048x2048 是单个纹理包的限制。你会看到 `game2.png`，可能还有 `game3.png` 或更多出现。

这对性能不好，这就是为什么基础游戏控制哪些类型的图像在一起进入一个纹理（+图集）。

这也适用于模组：创建不仅仅一个 Images 文件夹，而是创建多个，额外的文件夹命名为"Images.xyz"，其中 xyz 将成为额外纹理文件的文件名（所以不要同时使用 Images 和 Images.game——它们会冲突）。查看 Unciv 基础游戏以更好地了解它是如何工作的。

为了最小化纹理交换，尝试按它们在游戏中需要的情况对它们进行分组。你可以按文件夹分发，但在几个"Images.xyz"下有相同的子文件夹并在它们之间分发图像也可以工作。

有关示例，请参阅 [Alpha Frontier 模组](https://github.com/carriontrooper/Alpha-Frontier)。

模组根目录中的 `Atlases.json` 文件（大写'A'，不在 `Images` 或 `jsons` 中）控制加载哪些图集，这反过来控制读取哪些纹理（`.png`）文件。

此文件由内置打包器自动创建。默认情况下只读取 `game.atlas` 文件以保持向后兼容性。

如果你使用外部工具和多个图集，你需要自己维护此文件——它是一个简单的字符串 JSON 数组，每个都是不带 `.atlas` 扩展名的文件名（保存为 UTF-8 不带字节顺序标记）。

#### 更小的图标

许多图集可能是图标非常大的结果。

我们发现 100x100 像素对于大多数图标来说已经足够了。

#### 圆形图标

为了允许更快的图标渲染，这对性能有重大影响，你可以将 ["OtherIcons/circle.png"](https://yairm210.github.io/Unciv/android/Images.Icons/OtherIcons/Circle.png) 复制到：

- "ImprovementIcons/Circle.png" 用于改良
- "ResourceIcons/Circle.png" 用于资源
- "TechIcons/Circle.png" 用于科技
- "ConstructionIcons/Circle.png" 用于建筑和单位
- "StatIcons/Circle.png" 用于属性

### 纹理打包器设置

Unciv 内置的纹理打包器会在每个 `Images` 目录中查找 `TexturePacker.settings` 文件（不在 `jsons` 下）。

使用此文件，你可以调整打包器——例如控制像素插值过滤器。

它是 [Gdx TexturePacker.Settings](https://libgdx.com/wiki/tools/texture-packer#settings) 实例的 json。

如果你提供设置文件，默认值如 Gdx 文档中链接所示，但如果没有这样的文件，某些字段有不同的默认值。

要获得这些更改的默认值，请使用以下内容作为自定义 `TexturePacker.settings` 文件的基础：

```json
{
    "fast": true,
    "combineSubdirectories": true,
    "maxWidth": 2048,
    "maxHeight": 2048,
    "paddingX": 8,
    "paddingY": 8,
    "duplicatePadding": true,
    "filterMin": "MipMapLinearLinear",
    "filterMag": "MipMapLinearLinear"
}
```

（如果你的图集名称以"Icons"结尾，请将"filterMag"更改为"Linear"。）

### 纹理集编码

由于某些情况，请确保图集中映射的名称和路径使用**仅 ASCII**。加载器的所有部分并不强制严格的 UTF-8 使用，抱歉。

如果你不遵守此问题的症状：模组在中国 Windows 框上工作，但在西方框上不工作，反之亦然，或者模组在中国 Windows 框上工作，但在中国 Linux 框上不工作，反之亦然，或者模组在默认设置下的中国 Windows 框上工作，但在打开了"使用 unicode UTF-8 支持全球语言"的同一框上不工作。

这在技术上并不适用于使用多个图集时的图集名称本身（"Images.xyz"中的 xyz 部分），但我们为了一致性仍然建议相同的规则。

## 永久视听模组

以下章节描述了在模组***活动***时将起作用的可能性。

它要么为当前游戏选择（在创建新游戏期间，保存的游戏无法更改），意味着所有其规则和资源都将被使用。

**或者**它在模组管理器中标记为"永久视听模组"（你必须在"已安装"列中选择它以获取复选框）。

在这种情况下，只有图形和音频将处于活动状态，规则更改将被忽略（如果它包含任何），除非也使用第一种方式。

请注意，此功能包括来自所选模组的所有游戏中的图形或声音，甚至在安装模组之前开始的游戏。

重复：如果模组同时带来更改的规则和视听，"永久"功能将仅包括所有游戏上的媒体，要使用规则，你仍然需要为新游戏选择模组。

请注意，模组作者可以（并且经常应该）使用 [ModOptions](mod-file-structure/5-miscellaneous-json-files.md#modoptionsjson) [uniques](uniques.md#modoptions-uniques)控制复选框是否出现。

## 覆盖内置图形

如果模组提供与基础游戏中包含的图像具有相同名称和路径的图像（并且其图集是最新的），并且模组处于活动状态，则将使用模组的图形而不是内置图形。

例如，如果你在模组中包含名为"Images/OtherIcons/Link.png"的文件，你将覆盖 Civilopedia 中表示链接行的小链条图标。路径的第一部分与覆盖无关，它控制图集的一组文件将携带图像，但对于游戏中的选择只有路径的其余部分是相关的。因此，要覆盖"Images.Tech/TechIcons/Archery.png"，你可以将图像放置为"Images/TechIcons/Archery.png"，它将起作用，因为"TechIcons/Archery"部分是关键。

请注意，对于添加项目，你的图形应保持原始图像的大小和颜色选择，否则结果可能会令人惊讶，例如当游戏尝试为该图像着色时。

## 提供额外的图形

你需要为新元素提供图形——新单位需要它的图标，就像新文明一样。规则是：

- 图像文件的路径和名称需要符合规则：`Image[.AtlasName]/Type-specific/Objectname.png`（特定类型意味着"TechIcons"用于科技，"NationIcons"用于文明等。参见原版游戏文件夹。Objectname 是 json 中定义的精确名称，翻译之前。）
- 所有路径部分都是大小写敏感的。
- 单位像素精灵和[地形集](creating-a-custom-tileset.md)遵循特殊规则。
- 如果 `UnitIcons/<UnitName>.png` 不存在，我们将回退到 `UnitTypeIcons/<UnitType>.png` - 这允许为整个类型的单位设置单个图像，而无需摆弄每个单位
- 晋升可以命名为"[Unitname] ability"。在这种情况下，如果 `UnitIcons/Unitname.png` 存在，它将在 `UnitPromotionIcons/Unitname ability.png` 缺失时回退到该单位图标。
- 晋升可以命名为"Something I"（或" II"或" III"）。后缀将被删除并绘制为小星星，只加载基础 `UnitPromotionIcons/Something.png`。
- 晋升的特殊规则可以结合使用，例如"`[Warrior] ability III`"将回退到 Warrior 单位图标并在其上绘制 3 颗星星。

此外，还有一些图像类型，游戏具有显示能力但不自行提供图形，如下段所述：

### 添加自定义字体

你可以向游戏中添加自定义 `.ttf` 字体：将 `.ttf` 文件放在模组的 `/fonts/` 目录内。你添加的字体将在"选项-高级"选项卡的字体列表顶部可见并可选择为 `<fontname> (<modname>)`。

所有字体默认以 50 像素大小渲染，以后根据游戏需要进行缩放。目前字体在缩小时不进行多级映射。

### 覆盖特殊字符

EmojiIcons 子文件夹和其他文件夹中的纹理映射到特定代码点的字体中。游戏使用它们，可以在模组的任何文本中使用，并且可以通过模组纹理覆盖。

此外，某些代码点通常由所选系统字体提供，但有 EmojiIcons 名称，如果模组提供它们将覆盖字体字形（下表中标记为"可选"）。

请注意，为这些代码点提供的纹理**确实**尊重纵横比，它们**不**需要像许多内置图标那样的正方形！

| 符号 | 代码点 | Unicode 名称 | 纹理路径 | 可选 |
|:----:|:------:|:------------|:---------|:----:|
| ⛏ | U+26CF | 十字镐 | EmojiIcons/Automate | |
| ♪ | U+266A | 八分音符 | EmojiIcons/Culture | |
| ☠ | U+2620 | 骷髅标志 | EmojiIcons/Death | |
| ☮ | U+262E | 和平符号 | EmojiIcons/Faith | |
| ⚙ | U+2699 | 齿轮 | EmojiIcons/Production | |

完整列表请参见原始文档

### 添加奇观闪屏

你可以向模组添加奇观图像，当奇观完成时，它们将显示为标准图标的替代。图像需要是 .png，比例为 2:1，例如 200x100 像素。

将图像添加到 `/Images/WonderImages/`。它们需要根据 `Buildings.json` 中的名称字段命名，例如"Temple of Artemis.png"或"Stonehenge.png"

记住，为了与移动设备兼容，需要生成包含这些图像的新图集。

### 添加领袖肖像

基础游戏没有领袖肖像，但能够在问候、Civilopedia、外交屏幕或国家选择器中显示它们。模组可以通过将图像添加到 `/Images/LeaderIcons/` 来提供这些图像。文件名必须与 Nations.json 中定义的国家领袖名称完全对应，否则它们将被忽略。

如果它们是正方形，大小在 100x100 到 256x256 像素之间，并且在该区域内包含一些透明边框，则效果最好。

例如，[这里](https://github.com/yairm210/Unciv-leader-portrait-mod-example)是展示如何添加领袖肖像的模组，它可以补充基础游戏。

### 添加肖像

基础游戏使用平面图标，周围有彩色圆圈作为背景（例如，为了单位适应文明旗帜颜色），以表示实体，如：单位、建筑、科技、资源、改良、宗教、晋升、uniques、单位行动和 UI 中的国家。模组可以通过将图像添加到 `/Images/<entityType>Portraits/`（例如`/Images/BuildingPortraits/`、`/Images/ResourcePortraits/`等）来提供"肖像"——保持不变的静态图像——这些将在所有 UI 元素中使用（除了世界地图上的单位图标）。文件名必须与相应 JSONs（例如 Units.json、Buildings.json、TileResources.json 等）中定义的单位/建筑/科技/资源等名称完全对应，或者具有与它们应该替换的文件相同的名称，否则它们将被忽略。

如果模组提供'/Images/<entityType>Portraits/Background.png'图像，它们将用作相应肖像的背景而不是默认圆圈。肖像和背景如果是完整的 RGB 正方形，大小在 100x100 到 256x256 像素之间，并且在该区域内包含一些透明边框，则效果最好。

例如，[这里](https://github.com/vegeta1k95/Civ-5-Icons)是展示如何添加自定义肖像的模组，它可以补充基础游戏。

可用的 `<entityType>Portraits/` 包括：

* UnitPortraits
* BuildingPortraits
* TechPortraits
* ResourcePortraits
* ImprovementPortraits
* UnitPromotionPortraits
* UniquePortraits
* NationPortraits
* ReligionPortraits
* UnitActionPortraits

## 声音

标准值如下。声音本身可以在[这里](https://yairm210.github.io/Unciv/android/assets/sounds)找到。

- arrow, artillery, bombard, bombing, cannon, chimes, choir, click, coin, construction, elephant, fortify, gdrAttack, horse, jetgun, machinegun, metalhit, missile, nonmetalhit, nuke, paper, policy, promote, setup, shipguns, shot, slider, swap, tankshot, throw, torpedo, upgrade, whoosh

模组可以添加自己的声音，只要 attackSound 中的任何新值在模组的声音文件夹中有相应的声音文件，使用 mp3、ogg 或 wav 之一（文件扩展名必须与使用的编解码器匹配）。记住，名称是大小写敏感的。强烈建议小尺寸，Unciv 自己的声音使用 24kHz 联合立体声 8 位 VBR，大约 50-100kBps。

## 覆盖内置声音

这与图形的工作方式相同，除了不涉及图集。例如，你在模组中包含 sounds/Click.mp3，它将代替正常的点击声音播放。这些文件必须保持简短和小。未压缩时大于 1MB 的声音可能会在移动设备上中断或根本不播放。Unciv 试图标准化为 24kHz 采样率、联合立体声、低比特率 VBR（-128kbps）mp3。只识别 mp3 和 ogg 格式（但现有的 mp3 可以用 ogg 文件覆盖）。

## 提供额外音乐

模组 /music 文件夹中的声音文件（mp3 或 ogg）将被识别并在模组处于活动状态时使用。除了以下段落中描述的上下文特定音乐外，曲目将从所有可用曲目中随机播放（有一点偏差以避免近距离重复曲目）。没有覆盖——模组中的"thatched-villagers.mp3"将作为基础游戏提供给你下载的文件一起播放，并且具有相同的可能性。比特率或长度没有硬性技术限制，但大带宽需求可能会导致卡顿（"下一回合"结束时，就在世界地图更新之前，对于非常大的地图最可能导致这种情况）。

### 上下文敏感音乐：概述

音乐控制器通常会一首接一首地播放一个曲目，它们之间有一个暂停（可以在选项中更改）。当打开"离开游戏？"确认对话框时，播放将淡出并暂停，关闭时可以恢复。

游戏代码中有各种"触发器"启动新曲目的选择。新曲目将在必要时快速淡出当前播放曲目，然后开始播放。曲目选择涉及触发器提供的上下文和随机因素，并尝试在至少其他八个曲目播放之前不重复任何曲目。

模组可以提供自己的音乐文件夹，如果它们处于活动状态，其内容将与主音乐文件夹中的内容完全相同地对待。模组应该通过仔细选择文件名来控制其曲目的使用。模组开发人员可以查看控制台输出，以获取记录曲目选择以及触发参数或加载错误的消息。

有一个曲目是特殊的：Thatched Villagers（另见 credits.md）。如果音乐文件夹为空，游戏能够下载它，并且在使用音乐音量滑块时播放。如果出现某些问题，它也是后备曲目（但是，损坏的文件将关闭播放器，直到发生另一个触发器）。
